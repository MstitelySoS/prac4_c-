## Состав проекта

- `fork1.c` — демонстрация создания дочернего процесса с выводом PID родительского и дочернего процессов.  
- `fork2.c` — демонстрация осиротевшего дочернего процесса с использованием `sleep()`.  
- `myexec.c` — демонстрация замены процесса с помощью `execl()`.  
- `redirect.c` — демонстрация перенаправления стандартного вывода с использованием `dup2()` и `execvp()`.  
- `task.c` — программа с родительским и дочерним процессами, выводящими значения счётчика.  
- `Makefile` — автоматическая сборка всех программ.  
- `README.md` — описание проекта.  

## Назначение проекта

Практическая работа демонстрирует использование стандартных системных вызовов **API POSIX** для управления процессами в операционных системах UNIX/Linux.

В рамках работы реализованы и протестированы примеры использования следующих функций:

- создание процессов с помощью `fork()`;  
- определение идентификаторов текущего (`getpid()`) и родительского (`getppid()`) процессов;  
- приостановка выполнения процесса (`sleep()`);  
- замена текущего процесса другим исполняемым файлом (`execl()`, `execvp()`);  
- дублирование и перенаправление потоков ввода-вывода (`dup()`, `dup2()`);  
- взаимодействие родительского и дочернего процессов при выводе данных и синхронизации (`wait()`).
